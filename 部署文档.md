# 部署文档（生产 · CentOS）

Express：`/admin-api` + `frontend/dist` · SQLite · PM2 进程名 `wan-ai` · **勿开多实例**

---

## 0. 前置

需要：**Node.js LTS**（建议 [NodeSource](https://github.com/nodesource/distributions) 或 nvm，勿用系统自带过旧版本）、**Git**

**better-sqlite3 编译依赖 + Git**

```bash
# CentOS 7
yum groupinstall -y "Development Tools"
yum install -y python3 git

# CentOS 8 / Stream（把下面两行里的 yum 改成 dnf 再执行）
# dnf groupinstall -y "Development Tools"
# dnf install -y python3 git
```

```bash
npm i -g pm2
```

---

## 1. 首次部署（整段复制后只改仓库地址、目录、密码）

```bash
# 变量（按你环境改）
REPO='https://github.com/你的用户/你的仓库.git'
APP_DIR='/root/wan'

# 已有代码则：cd "$APP_DIR" && git pull，不要重复 clone
git clone "$REPO" "$APP_DIR"
cd "$APP_DIR"

npm ci || npm install
npm run build:prod

export NODE_ENV=production
export PORT=3000
export ADMIN_PASSWORD='你的强密码'
# export DATA_DIR=/root/wan-data
# export HOST=0.0.0.0

pm2 start ecosystem.config.cjs
pm2 save
pm2 startup
# ↑ 把终端里打印的那一行也执行掉（含 env PATH=... / systemctl 等）

pm2 status
pm2 logs wan-ai --lines 30
```

控制台安全组：**入站 TCP `3000`**。本机 **firewalld**（CentOS 常见）：

```bash
firewall-cmd --permanent --add-port=3000/tcp && firewall-cmd --reload
# systemctl enable firewalld --now   # 若未运行
```

验证：

```bash
curl -sI http://127.0.0.1:3000 | head -3
ss -tlnp | grep 3000
```

浏览器：`http://<公网IP>:3000/` · 接口同端口前缀 `/admin-api`

---

## 2. 改密码 / 改端口（换变量后要重来 PM2）

```bash
cd /root/wan   # 同 APP_DIR
export NODE_ENV=production
export PORT=3000
export ADMIN_PASSWORD='新密码'
pm2 delete wan-ai
pm2 start ecosystem.config.cjs
pm2 save
```

---

## 3. 更新代码

```bash
cd /root/wan
git pull
npm ci || npm install
npm run build:prod
pm2 restart wan-ai
pm2 save
```

---

## 4. 可选 Nginx

`proxy_pass http://127.0.0.1:3000`，配 `Host`、`X-Forwarded-Proto`；对外只开 80/443 时安全组改放通对应端口。

---

## 5. 排障

| 情况 | 命令或处理 |
|------|------------|
| `pm2 status` 没进程 | 执行过 `pm2 start ecosystem.config.cjs` 再 `pm2 save` |
| 本机 curl 通、外网不通 | 安全组 + 本机防火墙 · `ss` 看是否 `0.0.0.0:3000` |
| 换 SSH 窗口后变量丢了 | 重新 `export` 后 `pm2 delete wan-ai` → `start` → `save` |
