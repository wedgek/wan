## 1. 首次部署（整段复制后只改仓库地址、目录、密码）

```bash
# 变量（按你环境改）
REPO='https://github.com/你的用户/你的仓库.git'
APP_DIR='/root/wan'

# 已有代码则：cd "$APP_DIR" && git pull，不要重复 clone
git clone "$REPO" "$APP_DIR"
cd "$APP_DIR"

npm ci || npm install
npm run build:prod

export NODE_ENV=production
export PORT=3000
export ADMIN_PASSWORD='你的强密码'
# export DATA_DIR=/root/wan-data
# 直连公网调试：export HOST=0.0.0.0（默认即 0.0.0.0）
# 配好 Nginx 后：export HOST=127.0.0.1 且 export TRUST_PROXY=1，仅本机反代可访问 Node

pm2 start ecosystem.config.cjs
pm2 save
pm2 startup
# ↑ 把终端里打印的那一行也执行掉（含 env PATH=... / systemctl 等）

pm2 status
pm2 logs wan-ai --lines 30
```

控制台安全组：

- **不配 Nginx**：入站 TCP **`3000`**
- **配 Nginx（推荐）**：入站 **`80`** / **`443`**；**不要**再对公网开放 `3000`

### 1.1 火山引擎 TOS（服务端上传）与方舟视频（可选）

素材与附件上传走 **`POST /admin-api/storage/upload`**，由 Node 写入 **TOS**，浏览器不持有 AK/SK。

**密钥从哪来**：登录火山引擎控制台 → 右上角 **账号（或头像）** → **访问密钥** / **API 访问密钥**（或在 **访问控制 IAM** 里为用户创建密钥）。复制 **Access Key ID** 与 **Secret Key**（Secret 仅显示一次，请妥善保存）。

**本地开发（推荐）**：将仓库内 [`backend/.env.example`](backend/.env.example) 复制为 **`backend/.env`**，填写 `TOS_ACCESS_KEY`、`TOS_SECRET_KEY` 等；启动 `npm run dev` 时后端会自动加载（`backend/.env` 已加入 `.gitignore`，勿提交）。

**Linux / 服务器**：仍可用环境变量（与 `.env` 二选一或并存；已 `export` 的变量优先）：

```bash
export TOS_ACCESS_KEY='你的AccessKey'
export TOS_SECRET_KEY='你的SecretKey'
export TOS_REGION='cn-guangzhou'
export TOS_ENDPOINT='https://tos-cn-guangzhou.volces.com'
export TOS_BUCKET='wan-ai'
# 可选：自定义公网访问根（默认按 https://{bucket}.tos-{region}.volces.com 拼接）
# export TOS_PUBLIC_BASE_URL='https://wan-ai.tos-cn-guangzhou.volces.com'
# 单文件上限（MB），默认 512；需与 Nginx client_max_body_size 一致或更小
# export MAX_UPLOAD_MB=512
# export STORAGE_UPLOAD_RATE_PER_MIN=60
```

**TOS 跨域（控制台里 `fetch *.volces.com` 报 CORS）**：桶「公有读」只解决匿名能否拉到对象，**不会自动**带上 `Access-Control-Allow-Origin`。凡在页面里有脚本对直链做 `fetch`（含部分浏览器扩展注入的 `HEAD`/`fetch`），控制台会出现红字，而 `<img>` / `<video>` 仍可能正常显示。

在火山控制台 **对象存储 TOS** → 选中本仓库使用的桶 → **跨域访问**（或 **权限管理** 下的跨域相关入口，以控制台为准）新增一条规则，示例：

| 项 | 建议 |
|----|------|
| 来源 | `http://localhost:5173`（本地 Vite）；生产再加 `https://你的站点域名`（逐条添加，不用 `*` 除非你明确接受任意站） |
| Methods | `GET`、`HEAD` |
| 允许 Header | `*` |
| 暴露 Header（可选） | `Content-Length`、`Content-Range`、`ETag` |

保存后约 1 分钟内生效，再强刷页面；若仍有 `allUrl.js` 等**非本仓库文件名**的栈，可同时尝试**无痕窗口**排除扩展脚本。

**Seedance / 方舟（视频生成）**：与 TOS 的 **访问密钥（AK/SK）不是同一个**。在 [火山方舟](https://console.volcengine.com/ark) 控制台创建 **API Key**，写入环境变量 **`ARK_API_KEY`** 或 **`SEEDANCE_API_KEY`**（二选一即可，服务端读取，勿写进前端）。本地可放在 **`backend/.env`**，例如 `ARK_API_KEY=xxx`；改后需重启 Node。可选 `ARK_API_BASE`（默认 `https://ark.cn-beijing.volces.com/api/v3`）。详见 [`backend/services/seedanceClient.js`](backend/services/seedanceClient.js)。

**Seedance 2.0 多模态（对话创作 / 视频任务）**：在后台 **「视频模型配置」** 中为默认模型填写控制台 **推理接入点 ID**（`ep-` 开头，勿用占位符）。需上传「参考视频」时，对该模型打开 **「参考视频」** 开关。新库首次初始化会写入 2.0 默认占位行，**务必把 `api_model_id` 改成真实 ep**。同任务「图 + 参考视频」若需固定走另一接入点，可在 `backend/.env` 中设置 **`ARK_MULTIMODAL_MODEL_ID=ep-xxxx`**（可选）。

**二阶段（可选）**：可将方舟生成的视频 URL 再转存到 TOS，统一资产域名；当前默认仍保存方舟返回的 `result_url`。

本机 **firewalld**（CentOS 常见）：

```bash
# 仅 Node 直出
firewall-cmd --permanent --add-port=3000/tcp && firewall-cmd --reload
# 使用 Nginx 时改为（示例）
# firewall-cmd --permanent --add-service=http --add-service=https && firewall-cmd --reload
# systemctl enable firewalld --now   # 若未运行
```

验证：

```bash
curl -sI http://127.0.0.1:3000 | head -3
ss -tlnp | grep 3000
```

浏览器（未上 Nginx）：`http://<公网IP>:3000/` · 接口前缀 `/admin-api`

浏览器（已上 Nginx）：`http://<公网IP>/` · 接口仍为 `/admin-api`（同域）

## 4. Nginx（推荐）

**为什么**：静态资源由 Nginx 直出（`sendfile`、缓存头）通常比 Node 更省事；有 **HTTPS + HTTP/2** 时，浏览器可在单连接上并发多请求，缓解「一堆 JS/CSS 排队 Pending」的现象。仅 **HTTP:80** 时浏览器仍是 HTTP/1.1，但静态交给 Nginx 一般也更稳。

**思路**：`/` 与 `/assets/` 读磁盘上的 `frontend/dist`；仅 `/admin-api/` 反代到本机 `127.0.0.1:3000`。Node 只绑 **127.0.0.1**，公网不再直连 3000。

### 4.0 一键脚本（服务器上执行）

在仓库根目录（已 `git pull`、已 `npm run build:prod` 更佳）：

```bash
cd /root/wan   # 改成你的 APP_DIR
sudo APP_DIR=/root/wan bash scripts/setup-nginx.sh
```

脚本会：安装 Nginx（yum/dnf/apt）、写入 `/etc/nginx/conf.d/wan-ai.conf`、`nginx -t`、`systemctl restart nginx`；并在 SELinux 开启时尝试 `httpd_can_network_connect`。

然后务必做 **§4.3**（`HOST=127.0.0.1`、`TRUST_PROXY=1`、重启 PM2），云安全组放行 **80**。

### 4.1 安装（手动，可跳过若已用 4.0）

```bash
# CentOS 7/8 示例（包名因发行版略有差异）
yum install -y nginx   # 或 dnf install -y nginx
systemctl enable nginx --now
```

SELinux 若开启且反代报 502，可放开网络（按需）：

```bash
setsebool -P httpd_can_network_connect 1
```

### 4.2 配置项目根路径并载入站点

将 `deploy/nginx/wan-ai.conf` 里的 **`__APP_ROOT__`** 全部换成你的目录（与 `$APP_DIR` 一致，如 `/root/wan`），再复制到 Nginx：

```bash
APP_DIR=/root/wan   # 改成你的
sed "s|__APP_ROOT__|${APP_DIR}|g" "$APP_DIR/deploy/nginx/wan-ai.conf" | sudo tee /etc/nginx/conf.d/wan-ai.conf
sudo nginx -t && sudo systemctl reload nginx
```

（若与其它站点冲突，可自行改名 `conf.d` 文件名或调整 `listen` / `server_name`。）

### 4.3 让 Node 只监听本机 + 信任反代

```bash
cd "$APP_DIR"
export NODE_ENV=production
export PORT=3000
export HOST=127.0.0.1
export TRUST_PROXY=1
export ADMIN_PASSWORD='你的强密码'

pm2 delete wan-ai 2>/dev/null
pm2 start ecosystem.config.cjs
pm2 save
```

确认 Node 未监听在公网：

```bash
ss -tlnp | grep 3000   # 期望看到 127.0.0.1:3000
curl -sI http://127.0.0.1:3000/admin-api/...   # 仅本机可测
```

### 4.4 安全组与防火墙

- 云主机：**放行 80（及后续 443）**，**回收 3000 入站**（若曾开放）。
- `firewalld`：放行 `http`/`https`，见上文。

### 4.5 HTTPS + HTTP/2（可选）

配置文件末尾有注释示例：准备域名与证书（Let’s Encrypt / 云厂商证书），取消 `listen 443 ssl http2` 一段注释并填写 `ssl_certificate` 路径；可另加 80 → 443 跳转。配置后 `nginx -t && systemctl reload nginx`。

**说明**：「别的服务器没问题」常见于对端已用 **Nginx/Caddy**、或 **HTTPS/2**、或机器带宽与路由更好；不一定是应用逻辑差异。若仍有个别资源 Pending，可再开浏览器 Network 对照域名与状态码。

## 5. 排障

| 情况 | 命令或处理 |
|------|------------|
| `pm2 status` 没进程 | 执行过 `pm2 start ecosystem.config.cjs` 再 `pm2 save` |
| 本机 curl 通、外网不通 | 安全组 + 本机防火墙 · `ss` 看是否 `0.0.0.0:3000` |
| 换 SSH 窗口后变量丢了 | 重新 `export` 后 `pm2 delete wan-ai` → `start` → `save` |
